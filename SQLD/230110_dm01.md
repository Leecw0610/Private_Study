# 데이터모델링의 이해

1. 데이터 모델링의 특징; 추상화 / 단순화 / 명확

2. 데이터 모델링 단계;

   1. 개념적 모델링 :

   - 엔티티(업무상 관리가 필요한 관심사/ 명사형).속성 도출
   - 개념적 ERD작성

   2. 논리적 모델링 :

   - 정규화 (테이블 간에 중복된 데이터를 허용하지 않음),
     **무결성 (Integrity)** primary key 기능과 유사

   3. 물리적 모델링 :

   - 데이터베이스의 실제 구축 (테이블,인덱스,함수)
   - 성능, 보안, 가용성 고려 구축
